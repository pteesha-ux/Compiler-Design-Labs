# Lab 15 – Stack Storage Allocation Strategy

## AIM
To simulate stack storage allocation strategy used in compiler design.

## THEORY
Stack storage allocation assigns memory dynamically during program execution.

Variables are pushed into stack when allocated and popped when removed.

This strategy is used for local variables and function calls.

## ALGORITHM
1. Initialize stack memory.
2. Allocate memory addresses sequentially.
3. Push variables into stack.
4. Pop variables when deallocated.
5. Display stack contents.

## OUTPUT
Program simulates stack allocation and deallocation of variables.

## CONCLUSION
Stack storage allocation strategy implemented successfully.
