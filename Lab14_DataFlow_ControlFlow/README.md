# Lab 14 – Data Flow Analysis and Control Flow Graph

## AIM
To construct Control Flow Graph (CFG) and perform Data Flow Analysis for a given program.

## THEORY
Control Flow Graph (CFG) represents the execution flow between program statements.

Data Flow Analysis determines how variables are defined and used across program execution.

Reaching Definitions identifies where variables receive values during execution.

CFG helps in compiler optimization and program analysis.

## ALGORITHM
1. Represent program statements as nodes.
2. Identify control transfer between statements.
3. Construct CFG edges.
4. Identify variable definitions.
5. Display reaching definitions.

## OUTPUT
Program prints CFG structure and reaching definitions.

## CONCLUSION
Control Flow Graph and Data Flow Analysis were implemented successfully.
